## 四、完整开发计划（分阶段）

### P0 · 关键帧闭环 ✅ 已完成
首/尾帧、视频片段(Seedance/Sora/Veo)、配音、批量、ZIP 导出。

### P1 · 数据安全与一致性（优先，多集基石）
- [ ] **OPFS 视频存储**：大视频移出 IndexedDB（`videoStorageService`），避免容量爆炸
- [ ] **跨集资产同步**：`characterSyncService` + AssetSyncBanner（library.version ↔ ref.syncedVersion）
- [ ] **资产库 CRUD UI**：`CharacterLibrary` + link/unlink + CharacterLibraryPicker
- [ ] **AI 资产匹配**：`assetMatchService`（Jaccard+Dice 名称相似度自动复用）
- [ ] **项目导入/导出**：全量备份与迁移（seriesProject 三件套）

### P2 · 分镜工业化（创作力核心）
- [ ] **分镜指纹缓存**：`buildSceneReuseSignature`（改剧本某场景只重生成该场景，多集增量）
- [ ] **剧本手动编辑**：角色视觉 / 分镜角色列表 / 动作台词 inline 编辑
- [ ] **续写/改写**：continue/rewrite（含流式）
- [ ] **图推断风格**：`inferVisualStyleFromImage`
- [ ] **durationParser**：多格式时长解析替代固定下拉
- [ ] **九宫格分镜**：NineGridPreview + shotService 九宫格(4/6/9)

### P3 · 角色一致性增强
- [ ] **衣橱系统**：WardrobeModal + variations 生成
- [ ] **角色九宫格**：TurnaroundModal + 9 视角参考
- [ ] **批量角色提示词**：`generateAllCharacterPrompts`(6点结构)
- [ ] **参考图上传**：ImageUploadButton（用户自带定妆图）

### P4 · 质量与可控性
- [ ] **提示词模板系统**：`promptTemplateService` + `StagePrompts`(4类19字段 + 32运镜)
- [ ] **渲染日志**：`renderLogService` + RenderLogsModal/StatusPanel
- [ ] **质量评估**：V1规则 + V2 LLM 双版本
- [ ] **提示词预检**：`promptLintService`
- [ ] **提示词版本历史**：`promptVersionService`（回滚）
- [ ] **prompt 压缩**：长 prompt 三级管线

### P5 · 成片与体验
- [ ] **浏览器内混流成片**：canvas + MediaRecorder + AudioContext（合成单一 MP4/WebM）
- [ ] **Onboarding 引导**：5 步首次配置
- [ ] **GlobalAlert** 全局告警
- [ ] **媒体 CORS 代理**：`mediaFetchService`（签名 URL 跨域）

## 五、技术差异与设计决策

| 维度 | 参考项目 | 本项目 | 说明 |
| --- | --- | --- | --- |
| 存储 | IndexedDB + **OPFS**(视频) | 仅 IndexedDB(base64) | P1 需补 OPFS |
| 剧本管线 | structure→visuals→shots 三步+指纹缓存 | 两步合并 | P2 拆分+缓存 |
| 提示词 | 可编辑模板系统 + 32运镜 + 压缩 | 硬编码 | P4 模板化 |
| 成片 | canvas 混流单文件 | ZIP 分段 | P5 升级混流 |
| 模型 Key | 模型级>供应商>全局 三级 | 供应商>全局 两级 | P1 补模型级 |
| 多集同步 | 完整 syncStatus 机制 | 仅类型建模 | P1 实现 |
