# AI 漫剧平台 · 开发计划

> 本文档记录架构决策、已完成的第一轮交付，以及后续迭代路线。

## 1. 定位与目标

构建一个 **AI 漫剧工业化创作平台**：以「关键帧驱动」为核心，覆盖从故事文本到分镜成片的全流程，支持多集漫剧的项目化管理与跨集资产一致性。

## 2. 架构决策

| 决策点 | 选择 | 理由 |
| --- | --- | --- |
| 形态 | 纯前端 SPA | 数据本地化、隐私安全、零后端运维；参考项目验证可行 |
| 框架 | React 19 + TS + Vite | 类型安全、生态成熟、构建快 |
| 存储 | IndexedDB（`idb`） | 容纳 base64 图片/视频，容量远超 localStorage |
| AI 接入 | OpenAI 兼容适配器层 | 一套协议覆盖官方/聚合/Gemini，依赖倒置便于替换 |
| 多集模型 | 项目→季→集 + 共享资产库 | 漫剧天然多集；资产库保证角色场景跨集一致 |

## 3. 第一轮交付（已完成 ✅）

### 3.1 基础设施
- 工程脚手架：Vite 6 + React 19 + TS + Tailwind v4 + React Router 7
- 工业风明暗双主题（CSS 变量 + Tailwind `@theme inline`）
- IndexedDB 持久化层（项目/季/集/模型/KV，含级联删除）
- 领域工厂与通用工具（uid / base64 / JSON 提取 / 比例映射）

### 3.2 AI 适配器层（OpenAI 兼容）
- `chatAdapter`：对话补全 + 强制 JSON 解析（剧本推理）
- `imageAdapter`：图像生成（OpenAI 兼容 + Gemini 原生双形态）
- `videoAdapter`：提交 + 异步轮询通用框架（sora/veo）
- `audioAdapter`：TTS 语音合成
- 统一返回 base64 data url，规避链接过期

### 3.3 状态层
- `ModelContext`：模型配置加载/不可变更新/自动持久化 + 派生 AdapterContext
- `ProjectContext`：项目→季→集三级惰性加载 + 集 patch 高频编辑
- `ThemeContext`：主题偏好持久化

### 3.4 业务功能
- **项目管理 Dashboard**：项目列表、新建（风格/语言）、卡片入口
- **模型配置页**：供应商 CRUD、全局 Key、四类模型选择、默认比例
- **剧本阶段**：故事输入 → AI 拆解（角色/场景/道具/故事节拍/美术指导）→ 镜头规划 → 结构化展示
- **资产阶段**：角色定妆 / 场景概念 / 道具参考图，单个或批量生成，美术指导锚点注入
- **导演台**：镜头网格 + 关键帧首帧生成（场景+角色+动作联合驱动）
- **导出阶段**：时间轴预览（首帧缩略图 + 动作摘要）

### 3.5 验证
- `tsc -b` 类型检查通过
- `npm run build` 生产构建通过（gzip ≈ 95KB）

## 5. 关键工程约束

- **SOLID-DIP**：适配器仅依赖 `AdapterContext`，不耦合状态管理
- **DRY**：`VisualAsset` 公共基类统一角色/场景/道具
- **不可变更新**：所有 Context 变更返回新对象，便于持久化与 memo
- **失败可恢复**：生成状态持久化（pending/generating/completed/failed），刷新不丢失 loading
- **数据自洽**：ref→id 映射保证剧本拆解中跨实体引用一致
